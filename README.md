### 分支管理 

|            | master       | dev         | 
| ---------- | ------------ | ------------| 
| 本地仓库    |  --          |  源代码      |   
| Github     | 打包好的代码   |  源代码      | 
| 服务器仓库  |  打包好的代码   |  --        |

其中，Github和服务器仓库的master分支代码是本地仓库dev分支通过"hexo d"发布。Github仓库的dev分支是本地仓库dev分支通过"git push"推送。这样设计的目的是为了在Github仓库上master分支存放打包代码，dev存放源代码，开发者直接能在本地仓库dev分支开发、管理即可。
